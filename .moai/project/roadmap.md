## 현재 상태 요약

### 기술 스택

| 계층 | 기술 | 버전 |
|------|------|------|
| Backend | FastAPI + Uvicorn | FastAPI 0.135.x, Python 3.13 |
| Frontend | Next.js + React | Next.js 16.1.6, React 19.2.3 |
| Database | CockroachDB (pgvector) | PostgreSQL 호환, 벡터 검색 지원 |
| Cache | Upstash Redis | Redis 5.2+ |
| AI/LLM | Google Gemini 2.0 Flash | LangChain + google-generativeai |
| 벡터 검색 | pgvector + RRF 하이브리드 | Vector(1536) 임베딩 |
| 배포 | Fly.io (Backend), Vercel (Frontend) | Docker 기반 |
| Task Queue | Celery + Redis | 크롤링, 임베딩 비동기 처리 |
| UI | Tailwind CSS 4 + shadcn/ui | Base UI + CVA |

### 핵심 기능 (완료)

- 보험 약관 PDF 업로드 후 Gemini 2.0 Flash 실시간 분석, Q&A 채팅
- 약관 데이터베이스: 30개 보험사, ~4,555개 PDF 크롤링 완료
- RAG 기반 검색: pgvector 의미론적 검색 + tsvector 키워드 검색, RRF 하이브리드
- 분쟁 가이던스: 분쟁 감지, 판례 검색, 보상 확률 계산, 증거 조언
- 인증/OAuth: 이메일 + 카카오/네이버/구글 소셜 로그인
- B2B API: 조직 관리, API 키 발급, 사용량 추적, 대시보드
- 데이터 파이프라인: 크롤링 → 파싱 → 청킹 → 임베딩 자동화
- 보안: Rate Limiting, 보안 헤더, PII 암호화, Request ID 추적

### 현재 미완성 항목

- 손해보험 5개사 미수집 (한화, AXA, MG, NH, 롯데)
- **개별 보험 증권 등록 미구현** (핵심 기능, bodam-idea.md 명시)
- 프리미엄/과금 시스템 없음
- 마이페이지/대시보드 없음
- 프론트엔드 테스트 빈약

## Phase 1 (단기 1-4주): 핵심 가치 완성

| 태스크 | 목적 | 예상 공수 | 기대 효과 |
|--------|------|-----------|-----------|
| **개별 보험 증권 등록 MVP** | 사용자가 자신의 보험 증권을 등록하면 정확한 보상 금액 안내 | 2주 | 핵심 전환점: 일반 약관 Q&A → 개인화 보상 안내 서비스. 사용자 리텐션 3-5배 향상 |
| 미완료 손해보험 5개사 크롤링 | 한화, AXA, MG, NH, 롯데 약관 수집 | 1주 | 손해보험 커버리지 100% 달성 |
| 마이페이지/대시보드 | 등록 증권 목록, 보상 이력, 분석 히스토리 | 1주 | 서비스 재방문 유인, 사용자 데이터 축적 |
| 프론트엔드 테스트 보강 | Vitest + Testing Library 핵심 컴포넌트 | 3일 | 프로덕션 안정성 확보 |

> **핵심 논리**: "개별 증권 등록"은 보담의 핵심 차별화 요소. 약관 Q&A는 일반 정보 제공 수준이나, 개인 증권 기반 보상 안내는 실질적 가치 제공이며 유료 전환의 핵심 트리거.
